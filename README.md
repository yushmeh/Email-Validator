# Email Validator

Кроссплатформенный Python-модуль промышленного уровня для комплексной валидации email-адресов. Включает в себя строгую проверку синтаксиса на соответствие стандартам RFC и верификацию существования домена посредством обращения к DNS-серверам (поиск MX и A записей).

## Описание проекта

Модуль спроектирован с учетом принципов чистой архитектуры и SOLID. Логика синтаксического анализа полностью отделена от сетевых запросов. Для конфигурации поведения валидатора используется неизменяемый объект данных (Dataclass), позволяющий легко переопределять таймауты и типы проверяемых DNS-записей. Проект полностью типизирован и совместим с Python 3.14.

Поверх ядра бизнес-логики реализован высокопроизводительный асинхронный веб-сервер и интерактивный пользовательский интерфейс в стиле современных SaaS-платформ (Dark Mode, Glassmorphism).

## Установка

Склонируйте репозиторий или перенесите файлы модуля в ваш проект.

Убедитесь, что вы находитесь в корневом каталоге проекта EmailValidator и у вас активировано виртуальное окружение .venv.

Установите необходимые зависимости:

Bash
pip install -r requirements.txt
Примечание: содержимое файла requirements.txt должно включать:

Plaintext
fastapi>=0.110.0
uvicorn>=0.28.0
pydantic>=2.6.0
dnspython>=2.6.1
pytest>=8.0.0
Инструкция по запуску
Запуск веб-интерфейса (FastAPI + Uvicorn)
Для запуска локального сервера разработки выполните команду в терминале:

```Bash
python -m uvicorn main:app --reload
```

После этого откройте браузер по адресу: http://127.0.0.1:8000.
Вам откроется анимированный UI-интерфейс, отправляющий запросы к разработанному бэкенд-модулю без перезагрузки страницы.

## Запуск автоматических тестов (Pytest)

Для проверки стабильности работы ядра валидатора запустите тестовый фреймворк:

```Bash
python -m pytest -v
```

Все тесты используют моки (unittest.mock.patch), изолированы от реальной сети интернет и проверяют пограничные сценарии: от пустых строк до падения сетевых DNS-серверов.

## Спецификация API компонентов

POST /api/validate — принимает JSON-объект с проверяемым адресом: {"email": "user@domain.com"}.

Формат успешного ответа: {"email": "test@gmail.com", "valid": true, "reason": null}.

Формат ответа с ошибкой: {"email": "bad@domain", "valid": false, "reason": "Домен domain не имеет ни MX, ни A записей."}.
